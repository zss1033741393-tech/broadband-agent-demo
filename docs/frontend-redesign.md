# 前端改造方案

## 概述

现有前端逻辑大部分可复用（Zustand store、SSE 流解析、InsightEventParser、ECharts 渲染等），本次改造主要是**页面结构与视觉重设计**，新增页面 1（主界面）的全新布局，页面 2 延续现有 Workspace 逻辑并做适配。

---

## 页面结构

| 路由 | 页面 | 说明 |
|---|---|---|
| `/` 或 `/dashboard` | 页面 1：主界面 | 全新设计，含顶栏、左侧导航条、左侧面板、主内容区 |
| `/workspace` | 页面 2：Agent 工作台 | 现有 Workspace 页面，做适配调整 |

## 交互逻辑

### 1. 人工待决策事件卡片 → 跳转页面 2

- **触发**：点击左侧面板「人工带决策事件」区域的任意卡片（整张卡片均可点击）
- **行为**：路由跳转至 `/workspace`（页面 2），并携带该事件的上下文信息（如事件 ID、用户信息、问题描述）
- **页面 2 初始状态**：左侧对话框预填或显示该事件的问题描述，用户可直接确认发送或修改后发送
- **实现**：使用 `react-router-dom` 的 `useNavigate`，通过 `state` 或 query param 传递事件数据

### 2. 页面 2 对话框输入 → 触发 SSE 流程

- **复用现有逻辑**：与现有 Workspace 页面完全一致，输入问题后调用后端 SSE 接口，流式展示 Agent 执行过程
- **无需改动**：`workspaceStore.ts` 中的发送/接收/解析逻辑不变

### 3. 地图告警矩形交互（可选，后续扩展）

- 点击告警矩形可展开详情或跳转对应事件（当前阶段可不实现，预留 onClick 入口）

## 页面 2：Agent 工作台

### 与现有页面的差异

| 区域 | 现有状态 | 改造后 |
|---|---|---|
| 顶部导航栏 | 无 | 复用页面 1 的 TopBar 组件 |
| 左侧导航条 | 无 | 复用页面 1 的 Sidebar 组件 |
| 左侧对话面板 | 现有 ChatView（消息列表 + 输入框） | 保持不变，支持接收来自页面 1 的事件上下文预填 |
| 右侧图表画布 | 初始为空（EmptyState） | **初始展示一张预设图片**（如网络拓扑图或产品介绍图），收到第一个 render 事件后切换为图表渲染模式 |
| 分栏拖拽 | 已实现 | 保持不变 |

### 右侧初始图片

- **触发条件**：进入页面 2 且 `currentRenders` 为空时
- **内容**：预设一张图片（如网络拓扑示意图、产品 banner），后续可替换
- **实现**：修改 `RightPanel/index.tsx` 中的 `EmptyState` 组件，将空状态替换为图片展示
- **切换时机**：第一个 render 事件到达后，图片自动被图表内容替代

### 页面跳转数据结构

页面 1 事件卡片点击时，通过 `useNavigate` 携带以下 state 跳转到页面 2：

```typescript
interface EventCardNavState {
  eventId: string;
  userName: string;       // 如 "9801357468@139.gd"
  issue: string;          // 如 "直播卡顿，需Wi-Fi远程调优"
  prefillMessage?: string; // 预填到对话框的文本
}
```

页面 2 通过 `useLocation().state` 读取并预填对话框。
